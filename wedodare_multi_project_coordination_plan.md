# Wedodare Multi-Project Coordination & Alignment Plan

여러 개의 독립적인 프로젝트 폴더(루트 도메인, 블로그, 각종 기능 앱)를 개별적으로 관리할 때 발생할 수 있는 잠재적 문제점을 분석하고, 이를 가장 간단하고 일관되게 동기화할 수 있는 **마스터 시스템 구축 방안**을 제시합니다.

---

## 1. 발생 가능한 문제점 예측 및 원인 분석

독립 레포지토리 구조에서 개발을 진행할 때 마주치는 3가지 핵심 문제입니다.

| 문제 현상 | 원인 분석 | 예측되는 장애 상황 |
| :--- | :--- | :--- |
| **① 개발 컨텍스트 파편화**<br>(프로젝트마다 매번 같은 설명 반복) | 각 프로젝트 폴더마다 AI 에이전트나 개발자의 메모리가 공유되지 않음. | 매 프로젝트마다 Rewrite 설정 규칙, basePath 설정법, 에러 핸들링 규칙을 새로 주입해야 하여 피로감 누적 및 설정 누락 발생. |
| **② 디자인 및 스타일 불일치**<br>(각 사이트 이동 시 이질감 발생) | CSS 변수, 테마 설정(Tailwind v4), 폰트, 공통 UI(헤더/푸터)가 각 폴더별로 독립 구성됨. | 버튼의 곡률(Border Radius), 그림자(Box Shadow), 헤더의 높이나 반응형 중단점(Breakpoint)이 미세하게 달라 단일 웹사이트처럼 보이지 않음. |
| **③ 신규 프로젝트 생성 오버헤드**<br>(보일러플레이트 중복 구축) | 새 기능을 만들 때마다 환경 설정(shadcn/ui, Tailwind v4, lint 등)에 많은 시간 소요. | 프로젝트 시작 단계를 수동으로 반복 설정해야 하여 속도가 저하되고 불필요한 설정 불일치가 누적됨. |

---

## 2. 해결 방안: 마스터 시스템 구축 전략

이 문제를 해결하기 위해 **"하나의 기준 문서(Single Source of Truth)"**를 생성하여 모든 프로젝트에 일관되게 주입하는 전략을 제안합니다.

### 💡 방안 A. 마스터 가이드 파일 (`wedodare_system_spec.md`) 활용
개발 중인 루트 디렉토리에 시스템 명세서를 작성해두고, **다른 프로젝트 개발을 시작할 때 해당 파일을 AI에게 그대로 제공**합니다.
* AI에게 매번 구구절절 설명할 필요 없이, `"이 프로젝트를 첨부한 wedodare_system_spec.md 문서의 스펙에 맞춰 연동 설정 및 디자인 시스템을 적용해줘"` 한 마디로 모든 사전 설정과 디자인 톤앤매너를 일치시킬 수 있습니다.

### 💡 방안 B. Tailwind v4 CSS 변수 동기화
현재 블로그에 구현되어 있는 Tailwind v4 및 shadcn 기반의 `:root`, `.dark` CSS 변수를 명세화하여, 모든 프로젝트의 `globals.css`에 붙여넣기만 하면 자동으로 **민트색(Primary: `hsl(153 47% 49%)`) 테마와 OKLCH 기반의 다크 모드 톤이 100% 일치**하도록 설계합니다.

### 💡 방안 C. 통일된 헤더/푸터 UI 스펙 정의
* 각 프로젝트에서 헤더 컴포넌트를 독립적으로 그리되, 마스터 가이드에 **헤더의 높이, 마크업 구조, 아이콘 크기, 간격, 폰트 규격**을 명시합니다.
* 사용자가 링크를 클릭해 다른 서브디렉토리 프로젝트로 넘어가도 브라우저가 화면을 교체하는 시점에 이질감을 느끼지 않도록 마스터 스펙으로 고정합니다.

## 3. 마스터 시스템 명세서 (Wedodare System Spec)

아래 명세서를 파일로 보관해두고, **새로운 프로젝트를 생성하거나 기존 프로젝트를 연동할 때 AI 에이전트의 프롬프트로 입력**하시면 됩니다.

> [!TIP]
> 명세서 원본은 [wedodare_system_spec.md](file://./wedodare_system_spec.md) 파일로 따로 생성해 두었습니다. 새로운 프로젝트 작업 시 이 문서만 새 에이전트창에 드롭하여 사용하세요.

---

## 4. 향후 신규 프로젝트 시작 시 AI 요청 템플릿 (Prompt)

새로운 기능 프로젝트(예: `function3`)를 만들 때, AI 에이전트를 열어 아래 프롬프트를 복사-붙여넣기하여 지시하면 한 번에 설정을 끝낼 수 있습니다.

```markdown
# 역할 지정
너는 Wedodare 웹 플랫폼의 서브 프로젝트를 빌드하는 시니어 Next.js 개발자다.

# 요청 사항
새로운 기능 앱 프로젝트인 `[기능 이름, 예: pdf-converter]`를 다음 스펙에 맞춰 빌드/설정해라.

1. **아키텍처 설정**:
   - `next.config.js`에 `basePath: "/[기능경로, 예: pdf]"`를 추가해라.
   - 모든 정적 자산 및 이미지 참조 시 이 basePath를 고려해라.

2. **디자인 시스템 연동**:
   - 첨부된 `wedodare_system_spec.md` 파일에 정의된 `:root` 및 `.dark` CSS 변수를 `globals.css`에 100% 복사하여 적용해라.
   - Tailwind v4의 `@theme inline`을 활용하여 스타일 변수들을 바인딩해라.

3. **공통 헤더/푸터 UI 구현**:
   - `wedodare_system_spec.md`에 명시된 공통 글로벌 헤더 마크업 및 스타일링 규칙에 맞게 모바일 반응형 내비게이션 바를 구현해라.
   - 현재 활성화된 페이지 경로에 맞춰 헤더의 Active 상태(`text-primary`)가 제대로 처리되도록 작성해라.
```
